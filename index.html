Git Commands

Creating and Configuring Git

        
            create folder --> mkr foldername
            to go location --> cd foldername
            git config --global user.name "Harishkp77"
            git config --global user.email "shaarini_harish@outlook.com"
            to create git --> git init
            to check the files in folder --> dir
            we can add files in the folder, but they are untracked.
            to check --> git status
            staging the file --> git add index.html

Before Staging:

        
            D:\MyProjectLibrary\NewFolderTestGit>git status
            On branch master

            No commits yet

            Untracked files:
            (use "git add <file>..." to include in what will be committed)
                index.html

            nothing added to commit but untracked files present (use "git add" to track)

After Staging:

        
            D:\MyProjectLibrary\NewFolderTestGit>git add index.html
            D:\MyProjectLibrary\NewFolderTestGit>git status
            On branch master

            No commits yet

            Changes to be committed:
            (use "git rm --cached <file>..." to unstage)
                new file: index.html

Branching

        
            to add a branch --> git branch 'branchname' (e.g., git branch hello-world-images)
            to create a branch from master --> git branch hello-world-images * master
            after creating a branch, move the current workspace to the branch --> git checkout hello-world-images
            to check the branch status --> git status
            make changes, stage and commit for the branch
            to merge the branch into master, switch back to master --> git checkout master
            to merge the branch --> git merge hello-world-images
            delete the merged branch --> git branch -d hello-world-images

Merging Conflicts

        
            to fix errors in master without disturbing the branch --> git checkout -b emergency-fix
            (make changes, stage, and commit in emergency-fix branch)
            to merge emergency-fix into master, switch back to master --> git checkout master
            to merge the branch --> git merge emergency-fix
            delete the merged branch --> git branch -d emergency-fix
            (similarly, merge changes from another branch with conflicts)
            go to the editor and resolve conflicts, then stage the resolved file --> git add index.html
            check status and commit the merged changes --> git commit -m "merged with branch after fixing conflicts"

Github Integration

        
            add remote location to your local repo --> git remote add origin https://github.com/Harishkp77/gitdemo.git
            push the local repo to the remote repo --> git push --set-upstream origin master
            (when there are updates on the remote repo)
            update your local repo from remote --> git fetch origin
            check status and merge with remote/master --> git merge origin/master
            or simply use pull command --> git pull origin
